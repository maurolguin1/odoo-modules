Sale Order Line Quantity Recompute

Recompute sale quantity to approch real delivery quantity

Let starting with the following context: we want to buy product by unit, we want to manage product inventory by unit but we want to sell it by SQM and unit cannot be divide. The UOM settings for this context could be:

  • Unit:
    • Type: Smaller than the reference Unit of Measure
    • Rounding Precision: 1.00000
    • Ratio: 3.861003861
  • SQM:
    • Type: Reference Unit of Measure for this category
    • Rounding Precision: 0.00100

And the following settings for product:

  • Unit of Measure: Unit
  • Purchase Unit of Measure: Unit

When the seller change UOM for use SQM and sell 43 SQM ; it means we will charge 43 * 4.6 = 166.02316602 unit. But in fact as we setup the unit price with a rounding precision of 1.0 ; 167 piece will be send to the client.

This module try to reduce the loss by recomputing the surface (SQM) seized by the seller to approach as close as possible to the real surface that will be delivered to the client.
